# الحلم (تجريبي)

الحلم هو نظام تجميع الذاكرة في الخلفية ضمن `memory-core`.
يساعد OpenClaw على نقل الإشارات القصيرة المدى القوية إلى ذاكرة دائمة مع
الحفاظ على العملية قابلة للتفسير والمراجعة.

الحلم **اختياري** ومُعطّل افتراضيًا.

## ما الذي يكتبه الحلم

يحتفظ الحلم بنوعين من المخرجات:

- **حالة الآلة** في `memory/.dreams/` (مخزن الاستدعاء، إشارات المراحل، نقاط تحقق الإدخال، الأقفال).
- **مخرجات قابلة للقراءة البشرية** في `DREAMS.md` (أو `dreams.md` الموجود) وملفات تقارير مراحل اختيارية ضمن `memory/dreaming/<phase>/YYYY-MM-DD.md`.

تظل الترقية إلى الذاكرة طويلة المدى تكتب فقط إلى `MEMORY.md`.

## نموذج المراحل

يستخدم الحلم ثلاث مراحل تعاونية:

| المرحلة | الغرض                                    | الكتابة الدائمة     |
| ------- | ---------------------------------------- | ------------------- |
| الخفيفة | فرز وتجهيز المواد القصيرة المدى الحديثة  | لا                  |
| العميقة | تقييم المرشحين الدائمين وترقيتهم         | نعم (`MEMORY.md`)   |
| REM     | التأمل في السمات والأفكار المتكررة       | لا                  |

هذه المراحل هي تفاصيل تنفيذ داخلية وليست "أوضاعًا" منفصلة يضبطها المستخدم.

### المرحلة الخفيفة

تقوم المرحلة الخفيفة بإدخال إشارات الذاكرة اليومية الحديثة وآثار الاستدعاء، وإزالة التكرار منها،
وتجهيز الأسطر المرشحة.

- تقرأ من حالة الاستدعاء القصير المدى وملفات الذاكرة اليومية الحديثة.
- تكتب كتلة `## Light Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمنة.
- تسجل إشارات تعزيز لاستخدامها لاحقًا في الترتيب العميق.
- لا تكتب أبدًا إلى `MEMORY.md`.

### المرحلة العميقة

تقرر المرحلة العميقة ما الذي يصبح ذاكرة طويلة المدى.

- ترتب المرشحين باستخدام تقييم مرجّح وبوابات حدود دنيا.
- تتطلب اجتياز `minScore` و`minRecallCount` و`minUniqueQueries`.
- تعيد ترطيب المقاطع من الملفات اليومية الحية قبل الكتابة، لذلك يتم تخطي المقاطع القديمة أو المحذوفة.
- تلحق الإدخالات المُرقّاة إلى `MEMORY.md`.
- تكتب ملخص `## Deep Sleep` في `DREAMS.md` وتكتب اختياريًا إلى `memory/dreaming/deep/YYYY-MM-DD.md`.

### مرحلة REM

تستخرج مرحلة REM الأنماط والإشارات التأملية.

- تنشئ ملخصات للسمات والتأملات من آثار الذاكرة القصيرة المدى الحديثة.
- تكتب كتلة `## REM Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمنة.
- تسجل إشارات تعزيز REM المستخدمة في الترتيب العميق.
- لا تكتب أبدًا إلى `MEMORY.md`.

## يوميات الأحلام

يحتفظ الحلم أيضًا بسرد قصصي بعنوان **يوميات الأحلام** في `DREAMS.md`.
بعد أن تمتلك كل مرحلة مادة كافية، يشغّل `memory-core` بشكل أفضل جهد ممكن
دورة وكيل فرعي في الخلفية (باستخدام نموذج وقت التشغيل الافتراضي) ويلحق إدخالًا قصيرًا إلى اليوميات.

هذه اليوميات مخصصة للقراءة البشرية في واجهة Dreams، وليست مصدرًا للترقية.

## إشارات الترتيب العميق

يستخدم الترتيب العميق ست إشارات أساسية مرجّحة بالإضافة إلى تعزيز المراحل:

| الإشارة             | الوزن | الوصف                                              |
| ------------------- | ----- | -------------------------------------------------- |
| التكرار             | 0.24  | عدد إشارات الذاكرة القصيرة المدى التي راكمها الإدخال |
| الصلة               | 0.30  | متوسط جودة الاسترجاع للإدخال                       |
| تنوع الاستعلامات    | 0.15  | سياقات الاستعلام/اليوم المختلفة التي أظهرته       |
| الحداثة             | 0.15  | درجة حداثة متناقصة بمرور الوقت                     |
| التجميع             | 0.10  | قوة التكرار عبر أيام متعددة                        |
| الثراء المفاهيمي    | 0.06  | كثافة وسوم المفاهيم من المقطع/المسار               |

تضيف إصابات المرحلتين الخفيفة وREM تعزيزًا صغيرًا متناقصًا بمرور الوقت للحداثة من
`memory/.dreams/phase-signals.json`.

## الجدولة

عند التفعيل، يدير `memory-core` تلقائيًا مهمة cron واحدة لعملية
حلم كاملة. تشغّل كل عملية المراحل بالترتيب: الخفيفة -> REM -> العميقة.

سلوك الوتيرة الافتراضي:

| الإعداد              | الافتراضي  |
| -------------------- | ---------- |
| `dreaming.frequency` | `0 3 * * *` |

## البدء السريع

فعّل الحلم:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

فعّل الحلم مع وتيرة عملية مخصصة:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true,
            "timezone": "America/Los_Angeles",
            "frequency": "0 */6 * * *"
          }
        }
      }
    }
  }
}
```

## أمر الشرطة المائلة

```
/dreaming status
/dreaming on
/dreaming off
/dreaming help
```

## سير عمل CLI

استخدم ترقية CLI للمعاينة أو التطبيق اليدوي:

```bash
openclaw memory promote
openclaw memory promote --apply
openclaw memory promote --limit 5
openclaw memory status --deep
```

يستخدم `memory promote` اليدوي حدود المرحلة العميقة افتراضيًا ما لم يتم تجاوزها
بأعلام CLI.

اشرح سبب ترقية مرشح محدد أو عدم ترقيته:

```bash
openclaw memory promote-explain "router vlan"
openclaw memory promote-explain "router vlan" --json
```

عاين تأملات REM، والحقائق المرشحة، ومخرجات الترقية العميقة من دون
كتابة أي شيء:

```bash
openclaw memory rem-harness
openclaw memory rem-harness --json
```

## القيم الافتراضية الأساسية

توجد جميع الإعدادات ضمن `plugins.entries.memory-core.config.dreaming`.

| المفتاح     | الافتراضي  |
| ----------- | ---------- |
| `enabled`   | `false`    |
| `frequency` | `0 3 * * *` |

سياسة المراحل، والحدود، وسلوك التخزين هي تفاصيل تنفيذ داخلية
(وليست إعدادات موجهة للمستخدم).

راجع [مرجع إعدادات الذاكرة](/ar/reference/memory-config#dreaming-experimental)
للحصول على قائمة المفاتيح الكاملة.

## واجهة Dreams

عند التفعيل، تعرض علامة تبويب **Dreams** في Gateway ما يلي:

- حالة تفعيل الحلم الحالية
- الحالة على مستوى المرحلة ووجود العملية المُدارة
- أعداد الذاكرة القصيرة المدى، وطويلة المدى، والمُرقّاة اليوم
- توقيت التشغيل المجدول التالي
- قارئ يوميات أحلام قابل للتوسيع ومدعوم بواسطة `doctor.memory.dreamDiary`
