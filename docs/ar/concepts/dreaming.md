# Dreaming (تجريبي)

Dreaming هو نظام دمج الذاكرة في الخلفية في `memory-core`.
يساعد OpenClaw على نقل الإشارات القوية قصيرة الأمد إلى ذاكرة دائمة مع
الإبقاء على العملية قابلة للتفسير والمراجعة.

Dreaming **اختياري** وهو معطل افتراضيًا.

## ما الذي يكتبه Dreaming

يحتفظ Dreaming بنوعين من المخرجات:

- **حالة الآلة** في `memory/.dreams/` (مخزن الاسترجاع، وإشارات المراحل، ونقاط تحقق الإدخال، والأقفال).
- **مخرجات قابلة للقراءة البشرية** في `DREAMS.md` (أو `dreams.md` الموجود) وملفات تقارير المراحل الاختيارية تحت `memory/dreaming/<phase>/YYYY-MM-DD.md`.

لا تزال الترقية طويلة الأمد تكتب فقط إلى `MEMORY.md`.

## نموذج المراحل

يستخدم Dreaming ثلاث مراحل تعاونية:

| المرحلة | الغرض | الكتابة الدائمة |
| ----- | ----------------------------------------- | ----------------- |
| الخفيف | فرز وتجهيز المواد الحديثة قصيرة الأمد | لا |
| العميق | تقييم المرشحين الدائمين وترقيتهم | نعم (`MEMORY.md`) |
| REM   | التأمل في السمات والأفكار المتكررة | لا |

هذه المراحل هي تفاصيل تنفيذ داخلية وليست "أوضاعًا" منفصلة يضبطها المستخدم.

### المرحلة الخفيفة

تستوعب المرحلة الخفيفة إشارات الذاكرة اليومية الحديثة وآثار الاسترجاع، وتزيل
التكرار منها، وتجهز الأسطر المرشحة.

- تقرأ من حالة الاسترجاع قصيرة الأمد وملفات الذاكرة اليومية الحديثة.
- تكتب كتلة `## Light Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمّنة.
- تسجل إشارات التعزيز لاستخدامها لاحقًا في الترتيب العميق.
- لا تكتب أبدًا إلى `MEMORY.md`.

### المرحلة العميقة

تحدد المرحلة العميقة ما الذي يصبح ذاكرة طويلة الأمد.

- ترتب المرشحين باستخدام تقييم مرجّح وبوابات عتبة.
- تتطلب اجتياز `minScore` و`minRecallCount` و`minUniqueQueries`.
- تعيد ترطيب المقتطفات من الملفات اليومية الحية قبل الكتابة، لذلك يتم تخطي المقتطفات القديمة/المحذوفة.
- تُلحق الإدخالات المُرقّاة إلى `MEMORY.md`.
- تكتب ملخص `## Deep Sleep` في `DREAMS.md` وتكتب اختياريًا إلى `memory/dreaming/deep/YYYY-MM-DD.md`.

### مرحلة REM

تستخرج مرحلة REM الأنماط والإشارات التأملية.

- تنشئ ملخصات للسمات والتأملات من الآثار قصيرة الأمد الحديثة.
- تكتب كتلة `## REM Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمّنة.
- تسجل إشارات تعزيز REM المستخدمة في الترتيب العميق.
- لا تكتب أبدًا إلى `MEMORY.md`.

## يوميات الأحلام

يحتفظ Dreaming أيضًا بسرد **يوميات الأحلام** في `DREAMS.md`.
بعد أن تمتلك كل مرحلة مادة كافية، يشغّل `memory-core` دور وكيل فرعي في الخلفية
بأفضل جهد (باستخدام نموذج وقت التشغيل الافتراضي) ويُلحق إدخالًا قصيرًا في اليوميات.

هذه اليوميات مخصصة للقراءة البشرية في واجهة Dreams، وليست مصدرًا للترقية.

## إشارات الترتيب العميق

يستخدم الترتيب العميق ست إشارات أساسية مرجّحة بالإضافة إلى تعزيز المراحل:

| الإشارة | الوزن | الوصف |
| ------------------- | ------ | ------------------------------------------------- |
| التكرار | 0.24   | عدد الإشارات قصيرة الأمد التي راكمها الإدخال |
| الصلة | 0.30   | متوسط جودة الاسترجاع للإدخال |
| تنوع الاستعلامات | 0.15   | سياقات الاستعلام/اليوم المميزة التي أظهرته |
| الحداثة | 0.15   | درجة حداثة متناقصة مع الزمن |
| الدمج | 0.10   | قوة التكرار عبر عدة أيام |
| الثراء المفاهيمي | 0.06   | كثافة وسوم المفاهيم من المقتطف/المسار |

تضيف إصابات المرحلتين الخفيفة وREM دفعة صغيرة متناقصة مع الزمن إلى
`memory/.dreams/phase-signals.json`.

## الجدولة

عند التمكين، يدير `memory-core` تلقائيًا مهمة cron واحدة لاجتياح Dreaming
كامل. يشغّل كل اجتياح المراحل بالترتيب: الخفيف -> REM -> العميق.

سلوك الإيقاع الافتراضي:

| الإعداد | الافتراضي |
| -------------------- | ----------- |
| `dreaming.frequency` | `0 3 * * *` |

## البدء السريع

فعّل Dreaming:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

فعّل Dreaming مع إيقاع اجتياح مخصص:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true,
            "timezone": "America/Los_Angeles",
            "frequency": "0 */6 * * *"
          }
        }
      }
    }
  }
}
```

## أمر الشرطة المائلة

```
/dreaming status
/dreaming on
/dreaming off
/dreaming help
```

## سير عمل CLI

استخدم ترقية CLI للمعاينة أو التطبيق اليدوي:

```bash
openclaw memory promote
openclaw memory promote --apply
openclaw memory promote --limit 5
openclaw memory status --deep
```

يستخدم `memory promote` اليدوي عتبات المرحلة العميقة افتراضيًا ما لم يتم تجاوزها
باستخدام علامات CLI.

## القيم الافتراضية الأساسية

توجد جميع الإعدادات تحت `plugins.entries.memory-core.config.dreaming`.

| المفتاح | الافتراضي |
| ----------- | ----------- |
| `enabled`   | `false`     |
| `frequency` | `0 3 * * *` |

سياسة المراحل، والعتبات، وسلوك التخزين هي تفاصيل تنفيذ داخلية
(وليست إعدادات موجهة للمستخدم).

راجع [مرجع إعدادات الذاكرة](/ar/reference/memory-config#dreaming-experimental)
للاطلاع على القائمة الكاملة للمفاتيح.

## واجهة Dreams

عند التمكين، تعرض علامة تبويب **Dreams** في Gateway ما يلي:

- حالة تمكين Dreaming الحالية
- حالة على مستوى المراحل ووجود الاجتياح المُدار
- عدد العناصر قصيرة الأمد وطويلة الأمد والمُرقّاة اليوم
- توقيت التشغيل المجدول التالي
- قارئ يوميات أحلام قابل للتوسيع ومدعوم بواسطة `doctor.memory.dreamDiary`
