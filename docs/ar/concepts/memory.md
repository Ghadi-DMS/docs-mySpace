# نظرة عامة على الذاكرة

يتذكر OpenClaw الأشياء عن طريق كتابة **ملفات Markdown عادية** في
مساحة عمل الوكيل. لا "يتذكر" النموذج إلا ما يتم حفظه على القرص -- فلا توجد
أي حالة مخفية.

## كيف يعمل

لدى الوكيل مكانان لتخزين الذكريات:

- **`MEMORY.md`** -- الذاكرة طويلة المدى. الحقائق والتفضيلات والقرارات
  المستدامة. يتم تحميله عند بداية كل جلسة رسائل خاصة.
- **`memory/YYYY-MM-DD.md`** -- ملاحظات يومية. السياق الجاري والملاحظات.
  يتم تحميل ملاحظات اليوم والأمس تلقائيًا.

توجد هذه الملفات في مساحة عمل الوكيل (الافتراضي `~/.openclaw/workspace`).

<Tip>
إذا كنت تريد أن يتذكر الوكيل شيئًا ما، فما عليك إلا أن تطلب منه ذلك: "تذكر أنني
أفضل TypeScript." وسيكتبه في الملف المناسب.
</Tip>

## أدوات الذاكرة

يمتلك الوكيل أداتين للعمل مع الذاكرة:

- **`memory_search`** -- يعثر على الملاحظات ذات الصلة باستخدام البحث الدلالي، حتى
  عندما تختلف الصياغة عن الأصل.
- **`memory_get`** -- يقرأ ملف ذاكرة محددًا أو نطاقًا من الأسطر.

توفر الإضافة النشطة للذاكرة كلتا الأداتين (الافتراضي: `memory-core`).

## البحث في الذاكرة

عند تكوين مزود embeddings، يستخدم `memory_search` **البحث
الهجين** -- وهو يجمع بين التشابه المتجهي (المعنى الدلالي) ومطابقة الكلمات المفتاحية
(المصطلحات الدقيقة مثل المعرّفات ورموز الشيفرة). ويعمل هذا مباشرةً بمجرد أن يتوفر لديك
مفتاح API لأي مزود مدعوم.

<Info>
يكتشف OpenClaw مزود embeddings تلقائيًا من مفاتيح API المتاحة. وإذا
كان لديك مفتاح OpenAI أو Gemini أو Voyage أو Mistral مكوَّنًا، فسيتم
تمكين البحث في الذاكرة تلقائيًا.
</Info>

للحصول على تفاصيل حول كيفية عمل البحث، وخيارات الضبط، وإعداد المزود، راجع
[البحث في الذاكرة](/concepts/memory-search).

## التفريغ التلقائي للذاكرة

قبل أن يقوم [compaction](/concepts/compaction) بتلخيص المحادثة، يشغّل OpenClaw
دورًا صامتًا يذكّر الوكيل بحفظ السياق المهم في ملفات الذاكرة.
هذا مفعّل افتراضيًا -- ولا تحتاج إلى تكوين أي شيء.

<Tip>
يمنع تفريغ الذاكرة فقدان السياق أثناء compaction. وإذا كانت لدى الوكيل
حقائق مهمة في المحادثة لم تُكتب بعد إلى ملف، فسيتم حفظها تلقائيًا قبل
حدوث التلخيص.
</Tip>

## Dreaming (تجريبية)

Dreaming هي عملية دمج اختيارية في الخلفية للذاكرة. فهي تعيد زيارة
عمليات الاسترجاع قصيرة المدى من الملفات اليومية (`memory/YYYY-MM-DD.md`)،
وتقيّمها، ثم تروّج فقط العناصر المؤهلة إلى الذاكرة طويلة المدى (`MEMORY.md`).

وقد صُممت للحفاظ على كون الذاكرة طويلة المدى عالية الإشارة:

- **اختيارية**: معطلة افتراضيًا.
- **مجدولة**: عند التمكين، تدير `memory-core` المهمة المتكررة
  تلقائيًا.
- **محددة بعتبات**: يجب أن تتجاوز عمليات الترويج حدود التقييم، وتكرار الاسترجاع، وتنوع
  الاستعلامات.

للاطلاع على سلوك الأوضاع (`off` و`core` و`rem` و`deep`) وإشارات التقييم وخيارات
الضبط، راجع [Dreaming (تجريبية)](/concepts/memory-dreaming).

## CLI

```bash
openclaw memory status          # التحقق من حالة الفهرس والمزوّد
openclaw memory search "query"  # البحث من سطر الأوامر
openclaw memory index --force   # إعادة بناء الفهرس
```
