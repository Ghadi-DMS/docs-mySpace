# نظرة عامة على الذاكرة

يتذكر OpenClaw الأشياء من خلال كتابة **ملفات Markdown عادية** في
مساحة عمل الوكيل لديك. لا "يتذكر" النموذج إلا ما يتم حفظه على القرص --
فلا توجد حالة مخفية.

## كيف تعمل

يحتوي وكيلك على ثلاثة ملفات مرتبطة بالذاكرة:

- **`MEMORY.md`** -- الذاكرة طويلة الأمد. الحقائق الدائمة، والتفضيلات،
  والقرارات. يتم تحميله في بداية كل جلسة رسالة مباشرة.
- **`memory/YYYY-MM-DD.md`** -- ملاحظات يومية. سياق مستمر وملاحظات.
  يتم تحميل ملاحظات اليوم والأمس تلقائيًا.
- **`DREAMS.md`** (تجريبي، اختياري) -- يوميات الأحلام وملخصات
  جولات الحلم لمراجعة بشرية.

توجد هذه الملفات في مساحة عمل الوكيل (الافتراضي `~/.openclaw/workspace`).

<Tip>
إذا كنت تريد من وكيلك أن يتذكر شيئًا، فما عليك سوى أن تطلب منه ذلك: "تذكر أنني
أفضل TypeScript." وسيكتبه في الملف المناسب.
</Tip>

## أدوات الذاكرة

يمتلك الوكيل أداتين للعمل مع الذاكرة:

- **`memory_search`** -- يعثر على الملاحظات ذات الصلة باستخدام البحث الدلالي، حتى
  عندما تختلف الصياغة عن الأصل.
- **`memory_get`** -- يقرأ ملف ذاكرة محددًا أو نطاقًا من الأسطر.

يتم توفير كلتا الأداتين بواسطة plugin الذاكرة النشط (الافتراضي: `memory-core`).

## بحث الذاكرة

عند تكوين موفر للتضمين، يستخدم `memory_search` **بحثًا هجينًا**
-- يجمع بين التشابه المتجهي (المعنى الدلالي) ومطابقة الكلمات المفتاحية
(المصطلحات الدقيقة مثل المعرّفات ورموز الشيفرة). يعمل هذا مباشرةً بمجرد توفر
مفتاح API لأي موفر مدعوم.

<Info>
يكتشف OpenClaw موفر التضمين تلقائيًا من مفاتيح API المتاحة. إذا كان لديك
مفتاح OpenAI أو Gemini أو Voyage أو Mistral مُكوَّن، فسيتم تمكين بحث
الذاكرة تلقائيًا.
</Info>

للاطلاع على تفاصيل كيفية عمل البحث، وخيارات الضبط، وإعداد الموفر، راجع
[بحث الذاكرة](/ar/concepts/memory-search).

## التفريغ التلقائي للذاكرة

قبل أن يقوم [الضغط](/ar/concepts/compaction) بتلخيص محادثتك، يشغّل OpenClaw
دورًا صامتًا يذكّر الوكيل بحفظ السياق المهم في ملفات الذاكرة.
هذا مفعّل افتراضيًا -- ولا تحتاج إلى إعداد أي شيء.

<Tip>
يمنع تفريغ الذاكرة فقدان السياق أثناء الضغط. إذا كانت لدى وكيلك
حقائق مهمة في المحادثة لم تُكتب بعد إلى ملف، فسيتم حفظها تلقائيًا قبل
حدوث التلخيص.
</Tip>

## الحلم (تجريبي)

الحلم هو عملية تجميع اختيارية في الخلفية للذاكرة. فهو يجمع
الإشارات قصيرة الأمد، ويقيّم المرشحين، ويرقّي فقط العناصر المؤهلة إلى
الذاكرة طويلة الأمد (`MEMORY.md`).

وقد صُمم للحفاظ على ارتفاع قيمة الإشارة في الذاكرة طويلة الأمد:

- **اشتراك اختياري**: معطّل افتراضيًا.
- **مجدول**: عند التمكين، يدير `memory-core` تلقائيًا مهمة cron دورية واحدة
  لجولة حلم كاملة.
- **بعتبات**: يجب أن تجتاز الترقيات بوابات الدرجة، وتكرار الاستدعاء،
  وتنوع الاستعلام.
- **قابل للمراجعة**: تُكتب ملخصات المراحل وإدخالات اليوميات في `DREAMS.md`
  للمراجعة البشرية.

للاطلاع على سلوك المراحل، وإشارات التقييم، وتفاصيل يوميات الأحلام، راجع
[الحلم (تجريبي)](/concepts/dreaming).

## CLI

```bash
openclaw memory status          # تحقق من حالة الفهرس والموفر
openclaw memory search "query"  # ابحث من سطر الأوامر
openclaw memory index --force   # أعد بناء الفهرس
```
