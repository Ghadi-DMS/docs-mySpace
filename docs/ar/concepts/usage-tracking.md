# تتبع الاستخدام

## ما هو

- يسحب استخدام/حصة المزوّد مباشرة من نقاط نهاية الاستخدام الخاصة بهم.
- لا توجد تكاليف مقدّرة؛ فقط النوافذ التي يبلغ عنها المزوّد.
- يتم تطبيع خرج الحالة المقروء بشريًا إلى `X% left`، حتى عندما
  يبلّغ API من المصدر عن الحصة المستهلكة أو المتبقية أو عن أعداد خام فقط.
- يمكن لكل من `/status` على مستوى الجلسة و`session_status` الرجوع إلى أحدث
  إدخال استخدام في السجل النصي عندما تكون لقطة الجلسة المباشرة متناثرة. ويقوم
  هذا التراجع بملء عدادات الرموز/الذاكرة المؤقتة المفقودة، ويمكنه استعادة
  تسمية نموذج وقت التشغيل النشط، ويفضّل الإجمالي الأكبر الموجّه إلى prompt عندما تكون
  بيانات الجلسة الوصفية مفقودة أو أصغر. وتظل القيم المباشرة غير الصفرية الموجودة هي الأرجح.

## أين يظهر

- `/status` في الدردشات: بطاقة حالة غنية بالرموز التعبيرية تتضمن رموز الجلسة + التكلفة المقدّرة (مفتاح API فقط). ويظهر استخدام المزوّد لمزوّد **النموذج الحالي** عندما يكون متاحًا على شكل نافذة مطبّعة `X% left`.
- `/usage off|tokens|full` في الدردشات: تذييل استخدام لكل استجابة (تعرض OAuth الرموز فقط).
- `/usage cost` في الدردشات: ملخص تكلفة محلي مجمّع من سجلات جلسات OpenClaw.
- CLI: يطبع `openclaw status --usage` تفصيلاً كاملاً لكل مزوّد.
- CLI: يطبع `openclaw channels list` لقطة الاستخدام نفسها إلى جانب إعدادات المزوّد (استخدم `--no-usage` للتخطي).
- شريط قوائم macOS: قسم “Usage” ضمن Context ‏(فقط عند التوفر).

## المزوّدون + بيانات الاعتماد

- **Anthropic (Claude)**: رموز OAuth في ملفات تعريف المصادقة.
- **GitHub Copilot**: رموز OAuth في ملفات تعريف المصادقة.
- **Gemini CLI**: رموز OAuth في ملفات تعريف المصادقة.
  - يعود استخدام JSON إلى `stats`؛ ويتم تطبيع `stats.cached` إلى
    `cacheRead`.
- **OpenAI Codex**: رموز OAuth في ملفات تعريف المصادقة (يتم استخدام accountId عند وجوده).
- **MiniMax**: مفتاح API أو ملف تعريف مصادقة MiniMax OAuth. ويتعامل OpenClaw مع
  `minimax` و`minimax-cn` و`minimax-portal` على أنها سطح حصة MiniMax
  نفسه، ويفضّل MiniMax OAuth المخزن عند وجوده، ويعود بخلاف ذلك إلى
  `MINIMAX_CODE_PLAN_KEY` أو `MINIMAX_CODING_API_KEY` أو `MINIMAX_API_KEY`.
  وتعني حقول `usage_percent` / `usagePercent` الخام في MiniMax الحصة **المتبقية**،
  لذلك يعكسها OpenClaw قبل العرض؛ وتكون الأولوية للحقول المعتمدة على العد عند
  وجودها.
  - تأتي تسميات نوافذ خطة البرمجة من حقول الساعات/الدقائق الخاصة بالمزوّد عند
    وجودها، ثم تعود إلى المجال `start_time` / `end_time`.
  - إذا أعادت نقطة نهاية خطة البرمجة `model_remains`، فإن OpenClaw يفضّل
    إدخال نموذج الدردشة، ويستمد تسمية النافذة من الطوابع الزمنية عند غياب الحقول الصريحة
    `window_hours` / `window_minutes`، ويضمّن اسم النموذج
    في تسمية الخطة.
- **Xiaomi MiMo**: مفتاح API عبر البيئة/الإعدادات/مخزن المصادقة (`XIAOMI_API_KEY`).
- **z.ai**: مفتاح API عبر البيئة/الإعدادات/مخزن المصادقة.

يتم إخفاء الاستخدام عندما لا يمكن حل مصادقة صالحة لاستخدام المزوّد. ويمكن للمزوّدين
توفير منطق مصادقة استخدام خاص بـ plugin؛ وإلا يعود OpenClaw إلى
مطابقة بيانات اعتماد OAuth/مفتاح API من ملفات تعريف المصادقة، أو متغيرات البيئة،
أو الإعدادات.
