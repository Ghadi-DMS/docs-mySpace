# نظرة عامة على الإعداد الأولي

يحتوي OpenClaw على مسارين للإعداد الأولي. يقوم كلاهما بتهيئة المصادقة، وGateway،
وقنوات الدردشة الاختيارية — ويختلفان فقط في طريقة تفاعلك مع عملية الإعداد.

## ما المسار الذي ينبغي أن أستخدمه؟

|                | الإعداد الأولي عبر CLI                 | الإعداد الأولي عبر تطبيق macOS |
| -------------- | -------------------------------------- | ------------------------------ |
| **المنصات**    | macOS وLinux وWindows (أصليًا أو WSL2) | macOS فقط                      |
| **الواجهة**    | معالج طرفية                            | واجهة موجهة داخل التطبيق       |
| **الأفضل لـ**  | الخوادم، والبيئات بلا واجهة، والتحكم الكامل | أجهزة Mac المكتبية، والإعداد المرئي |
| **الأتمتة**    | `--non-interactive` للسكريبتات         | يدوي فقط                       |
| **الأمر**      | `openclaw onboard`                     | شغّل التطبيق                   |

يجب على معظم المستخدمين البدء باستخدام **الإعداد الأولي عبر CLI** — فهو يعمل في كل مكان ويمنحك
أكبر قدر من التحكم.

## ما الذي يقوم الإعداد الأولي بتهيئته

بغض النظر عن المسار الذي تختاره، يقوم الإعداد الأولي بإعداد:

1. **موفر النموذج والمصادقة** — مفتاح API، أو OAuth، أو رمز إعداد للموفر الذي اخترته
2. **مساحة العمل** — دليل لملفات الوكيل، وقوالب التمهيد، والذاكرة
3. **Gateway** — المنفذ، وعنوان الربط، ووضع المصادقة
4. **القنوات** (اختياري) — قنوات الدردشة المدمجة والمجمعة مثل
   BlueBubbles، وDiscord، وFeishu، وGoogle Chat، وMattermost، وMicrosoft Teams،
   وTelegram، وWhatsApp، وغيرها
5. **الخدمة الخلفية** (اختياري) — خدمة تعمل في الخلفية بحيث يبدأ Gateway تلقائيًا

## الإعداد الأولي عبر CLI

شغّله في أي طرفية:

```bash
openclaw onboard
```

أضف `--install-daemon` لتثبيت الخدمة الخلفية أيضًا في خطوة واحدة.

المرجع الكامل: [الإعداد الأولي (CLI)](/ar/start/wizard)
توثيق أمر CLI: [`openclaw onboard`](/cli/onboard)

## الإعداد الأولي عبر تطبيق macOS

افتح تطبيق OpenClaw. سيرشدك معالج التشغيل الأول عبر الخطوات نفسها
باستخدام واجهة مرئية.

المرجع الكامل: [الإعداد الأولي (تطبيق macOS)](/start/onboarding)

## الموفّرون المخصصون أو غير المدرجين

إذا لم يكن موفرك مدرجًا في الإعداد الأولي، فاختر **Custom Provider** وأدخل:

- وضع توافق API ‏(متوافق مع OpenAI، أو متوافق مع Anthropic، أو اكتشاف تلقائي)
- عنوان URL الأساسي ومفتاح API
- معرّف النموذج واسمًا مستعارًا اختياريًا

يمكن أن تتعايش عدة نقاط نهاية مخصصة — إذ يحصل كل منها على معرّف نقطة نهاية خاص بها.
