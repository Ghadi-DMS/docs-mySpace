# تسجيل الدخول إلى المتصفح + النشر على X/Twitter

## تسجيل الدخول اليدوي (موصى به)

عندما يتطلب موقع ما تسجيل الدخول، **سجّل الدخول يدويًا** في ملف تعريف المتصفح على **المضيف** (متصفح openclaw).

**لا** تعطِ بيانات اعتمادك للنموذج. غالبًا ما تؤدي عمليات تسجيل الدخول المؤتمتة إلى تشغيل وسائل الحماية من الروبوتات وقد تتسبب في قفل الحساب.

للعودة إلى مستندات المتصفح الرئيسية: [المتصفح](/tools/browser).

## أي ملف تعريف Chrome يُستخدم؟

يتحكم OpenClaw في **ملف تعريف Chrome مخصص** (اسمه `openclaw`، مع واجهة بلون برتقالي). وهذا منفصل عن ملف تعريف المتصفح الذي تستخدمه يوميًا.

بالنسبة إلى استدعاءات أداة المتصفح الخاصة بالوكيل:

- الخيار الافتراضي: يجب أن يستخدم الوكيل متصفح `openclaw` المعزول الخاص به.
- استخدم `profile="user"` فقط عندما تكون جلسات تسجيل الدخول الحالية مهمة ويكون المستخدم أمام الكمبيوتر للنقر/الموافقة على أي مطالبة إرفاق.
- إذا كان لديك عدة ملفات تعريف لمتصفح المستخدم، فحدّد ملف التعريف صراحةً بدلًا من التخمين.

هناك طريقتان سهلتان للوصول إليه:

1. **اطلب من الوكيل فتح المتصفح** ثم سجّل الدخول بنفسك.
2. **افتحه عبر CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

إذا كان لديك عدة ملفات تعريف، مرّر `--browser-profile <name>` (الافتراضي هو `openclaw`).

## X/Twitter: المسار الموصى به

- **القراءة/البحث/سلاسل المحادثات:** استخدم متصفح **المضيف** (مع تسجيل دخول يدوي).
- **نشر التحديثات:** استخدم متصفح **المضيف** (مع تسجيل دخول يدوي).

## العزل + الوصول إلى متصفح المضيف

تكون جلسات المتصفح المعزولة **أكثر عرضة** لتشغيل اكتشاف الروبوتات. بالنسبة إلى X/Twitter (والمواقع الصارمة الأخرى)، يُفضّل استخدام متصفح **المضيف**.

إذا كان الوكيل معزولًا، فإن أداة المتصفح تستخدم العزل افتراضيًا. للسماح بالتحكم في المضيف:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

ثم استهدف متصفح المضيف:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

أو عطّل العزل للوكيل الذي ينشر التحديثات.
