# إنشاء الصور

تتيح أداة `image_generate` للوكيل إنشاء الصور وتعديلها باستخدام المزوّدين المُعدّين لديك. يتم تسليم الصور المُنشأة تلقائيًا كمرفقات وسائط في رد الوكيل.

<Note>
لا تظهر الأداة إلا عند توفر مزوّد واحد على الأقل لإنشاء الصور. إذا لم ترَ `image_generate` ضمن أدوات الوكيل لديك، فقم بإعداد `agents.defaults.imageGenerationModel` أو اضبط مفتاح API لمزوّد.
</Note>

## البدء السريع

1. اضبط مفتاح API لمزوّد واحد على الأقل (على سبيل المثال `OPENAI_API_KEY` أو `GEMINI_API_KEY`).
2. اضبط نموذجك المفضل اختياريًا:

```json5
{
  agents: {
    defaults: {
      imageGenerationModel: {
        primary: "openai/gpt-image-1",
      },
    },
  },
}
```

3. اطلب من الوكيل: _"أنشئ صورة لتميمة سرطان بحر ودودة."_

يستدعي الوكيل `image_generate` تلقائيًا. لا حاجة إلى إدراجه في قائمة السماح للأدوات — فهو مفعّل افتراضيًا عند توفر مزوّد.

## المزوّدون المدعومون

| المزوّد | النموذج الافتراضي                 | دعم التعديل                        | مفتاح API                                              |
| ------- | --------------------------------- | ---------------------------------- | ------------------------------------------------------ |
| OpenAI  | `gpt-image-1`                     | نعم (حتى 5 صور)                    | `OPENAI_API_KEY`                                       |
| Google  | `gemini-3.1-flash-image-preview`  | نعم                                | `GEMINI_API_KEY` أو `GOOGLE_API_KEY`                   |
| fal     | `fal-ai/flux/dev`                 | نعم                                | `FAL_KEY`                                              |
| MiniMax | `image-01`                        | نعم (مرجع للموضوع)                 | `MINIMAX_API_KEY` أو MiniMax OAuth (`minimax-portal`)  |
| ComfyUI | `workflow`                        | نعم (صورة واحدة، وفق إعداد سير العمل) | `COMFY_API_KEY` أو `COMFY_CLOUD_API_KEY` للسحابة       |
| Vydra   | `grok-imagine`                    | لا                                 | `VYDRA_API_KEY`                                        |

استخدم `action: "list"` لفحص المزوّدين والنماذج المتاحة في وقت التشغيل:

```
/tool image_generate action=list
```

## معلمات الأداة

| المعلمة      | النوع    | الوصف                                                                                 |
| ------------ | -------- | ------------------------------------------------------------------------------------- |
| `prompt`     | string   | موجّه إنشاء الصورة (مطلوب لـ `action: "generate"`)                                   |
| `action`     | string   | `"generate"` (الافتراضي) أو `"list"` لفحص المزوّدين                                  |
| `model`      | string   | تجاوز للمزوّد/النموذج، مثل `openai/gpt-image-1`                                      |
| `image`      | string   | مسار صورة مرجعية واحدة أو URL لوضع التعديل                                            |
| `images`     | string[] | صور مرجعية متعددة لوضع التعديل (حتى 5)                                                |
| `size`       | string   | تلميح للحجم: `1024x1024` أو `1536x1024` أو `1024x1536` أو `1024x1792` أو `1792x1024` |
| `aspectRatio`| string   | نسبة الأبعاد: `1:1` أو `2:3` أو `3:2` أو `3:4` أو `4:3` أو `4:5` أو `5:4` أو `9:16` أو `16:9` أو `21:9` |
| `resolution` | string   | تلميح للدقة: `1K` أو `2K` أو `4K`                                                     |
| `count`      | number   | عدد الصور المطلوب إنشاؤها (1–4)                                                       |
| `filename`   | string   | تلميح لاسم ملف الإخراج                                                                |

لا تدعم جميع المزوّدات جميع المعلمات. تمرّر الأداة ما يدعمه كل مزوّد، وتتجاهل الباقي، وتبلّغ عن التجاوزات المُسقطة في نتيجة الأداة.

## الإعداد

### اختيار النموذج

```json5
{
  agents: {
    defaults: {
      imageGenerationModel: {
        primary: "openai/gpt-image-1",
        fallbacks: ["google/gemini-3.1-flash-image-preview", "fal/fal-ai/flux/dev"],
      },
    },
  },
}
```

### ترتيب اختيار المزوّد

عند إنشاء صورة، يحاول OpenClaw المزوّدين بهذا الترتيب:

1. معلمة **`model`** من استدعاء الأداة (إذا حدّد الوكيل واحدة)
2. **`imageGenerationModel.primary`** من الإعداد
3. **`imageGenerationModel.fallbacks`** بالترتيب
4. **الاكتشاف التلقائي** — يستخدم افتراضيات المزوّدات المدعومة بالمصادقة فقط:
   - المزوّد الافتراضي الحالي أولًا
   - مزوّدو إنشاء الصور المسجّلون المتبقون بترتيب معرّف المزوّد

إذا فشل أحد المزوّدين (خطأ مصادقة أو حد المعدل أو غير ذلك)، تتم تجربة المرشح التالي تلقائيًا. إذا فشل الجميع، فسيتضمن الخطأ تفاصيل من كل محاولة.

ملاحظات:

- الاكتشاف التلقائي يراعي حالة المصادقة. لا يدخل الافتراضي الخاص بمزوّد إلى قائمة المرشحين
  إلا عندما يتمكن OpenClaw فعليًا من مصادقة ذلك المزوّد.
- استخدم `action: "list"` لفحص المزوّدين المسجّلين حاليًا،
  ونماذجهم الافتراضية، وتلميحات متغيرات البيئة الخاصة بالمصادقة.

### تعديل الصور

تدعم OpenAI وGoogle وfal وMiniMax وComfyUI تعديل الصور المرجعية. مرّر مسار صورة مرجعية أو URL:

```
"أنشئ نسخة بألوان مائية من هذه الصورة" + image: "/path/to/photo.jpg"
```

تدعم OpenAI وGoogle حتى 5 صور مرجعية عبر المعلمة `images`. أما fal وMiniMax وComfyUI فتدعم صورة واحدة.

يتوفر إنشاء الصور في MiniMax عبر مساري مصادقة MiniMax المضمّنين كليهما:

- `minimax/image-01` لإعدادات مفتاح API
- `minimax-portal/image-01` لإعدادات OAuth
