# إنشاء الفيديو

يمكن لوكلاء OpenClaw إنشاء مقاطع فيديو من المطالبات النصية أو الصور المرجعية أو مقاطع الفيديو الموجودة. يتم دعم اثنتي عشرة واجهة مزود، ولكل منها خيارات نماذج وأنماط إدخال ومجموعات ميزات مختلفة. يختار الوكيل المزود المناسب تلقائيًا استنادًا إلى التكوين الخاص بك ومفاتيح API المتاحة.

<Note>
لا تظهر أداة `video_generate` إلا عند توفر مزود واحد على الأقل لإنشاء الفيديو. إذا لم تكن تراها ضمن أدوات الوكيل لديك، فاضبط مفتاح API لمزود أو قم بتكوين `agents.defaults.videoGenerationModel`.
</Note>

## البدء السريع

1. اضبط مفتاح API لأي مزود مدعوم:

```bash
export GEMINI_API_KEY="your-key"
```

2. ثبّت نموذجًا افتراضيًا اختياريًا:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "google/veo-3.1-fast-generate-preview"
```

3. اطلب من الوكيل:

> أنشئ فيديو سينمائيًا مدته 5 ثوانٍ لكركند ودود يركب الأمواج عند غروب الشمس.

يستدعي الوكيل `video_generate` تلقائيًا. لا حاجة إلى قائمة سماح للأداة.

## ما الذي يحدث عند إنشاء فيديو

إنشاء الفيديو غير متزامن. عندما يستدعي الوكيل `video_generate` في جلسة:

1. يرسل OpenClaw الطلب إلى المزود ويعيد فورًا معرّف مهمة.
2. يعالج المزود المهمة في الخلفية (عادةً من 30 ثانية إلى 5 دقائق بحسب المزود والدقة).
3. عندما يصبح الفيديو جاهزًا، يوقظ OpenClaw الجلسة نفسها بحدث إكمال داخلي.
4. ينشر الوكيل الفيديو النهائي مرة أخرى في المحادثة الأصلية.

أثناء تنفيذ المهمة، تعيد استدعاءات `video_generate` المكررة في الجلسة نفسها حالة المهمة الحالية بدلًا من بدء عملية إنشاء أخرى. استخدم `openclaw tasks list` أو `openclaw tasks show <taskId>` للتحقق من التقدم من خلال CLI.

خارج عمليات تشغيل الوكيل المعتمدة على الجلسات (على سبيل المثال، الاستدعاءات المباشرة للأدوات)، تعود الأداة إلى الإنشاء المضمن وتعيد مسار الوسائط النهائي في الدورة نفسها.

## معلمات الأداة

### مطلوبة

| المعلمة | النوع  | الوصف                                                                         |
| ------- | ------ | ----------------------------------------------------------------------------- |
| `prompt`  | string | وصف نصي للفيديو المراد إنشاؤه (مطلوب عند `action: "generate"`) |

### مدخلات المحتوى

| المعلمة | النوع    | الوصف                               |
| ------- | -------- | ----------------------------------- |
| `image`   | string   | صورة مرجعية واحدة (مسار أو URL)     |
| `images`  | string[] | صور مرجعية متعددة (حتى 5)           |
| `video`   | string   | فيديو مرجعي واحد (مسار أو URL)      |
| `videos`  | string[] | مقاطع فيديو مرجعية متعددة (حتى 4)   |

### عناصر التحكم في النمط

| المعلمة         | النوع   | الوصف                                                                    |
| ---------------- | ------- | ------------------------------------------------------------------------ |
| `aspectRatio`     | string  | `1:1` أو `2:3` أو `3:2` أو `3:4` أو `4:3` أو `4:5` أو `5:4` أو `9:16` أو `16:9` أو `21:9`  |
| `resolution`      | string  | `480P` أو `720P` أو `1080P`                                             |
| `durationSeconds` | number  | المدة المستهدفة بالثواني (تُقرّب إلى أقرب قيمة يدعمها المزود)          |
| `size`            | string  | تلميح للحجم عندما يدعمه المزود                                          |
| `audio`           | boolean | تمكين الصوت المُنشأ عند الدعم                                           |
| `watermark`       | boolean | تبديل العلامة المائية الخاصة بالمزود عند الدعم                          |

### متقدمة

| المعلمة  | النوع  | الوصف                                            |
| ---------- | ------ | ------------------------------------------------ |
| `action`   | string | `"generate"` (افتراضي) أو `"status"` أو `"list"` |
| `model`    | string | تجاوز المزود/النموذج (مثل `runway/gen4.5`)       |
| `filename` | string | تلميح لاسم ملف الإخراج                           |

لا تدعم جميع المزودات جميع المعلمات. يتم تجاهل التجاوزات غير المدعومة على أساس أفضل جهد، ويتم الإبلاغ عنها كتحذيرات في نتيجة الأداة. أما حدود القدرات الصارمة (مثل كثرة المدخلات المرجعية) فتفشل قبل الإرسال.

## الإجراءات

- **generate** (افتراضي) -- أنشئ فيديو من المطالبة المحددة والمدخلات المرجعية الاختيارية.
- **status** -- تحقّق من حالة مهمة الفيديو قيد التنفيذ للجلسة الحالية دون بدء عملية إنشاء أخرى.
- **list** -- اعرض المزودين والنماذج المتاحة وقدراتها.

## اختيار النموذج

عند إنشاء فيديو، يحدد OpenClaw النموذج بهذا الترتيب:

1. **معلمة الأداة `model`** -- إذا حدّدها الوكيل في الاستدعاء.
2. **`videoGenerationModel.primary`** -- من التكوين.
3. **`videoGenerationModel.fallbacks`** -- تتم تجربتها بالترتيب.
4. **الاكتشاف التلقائي** -- يستخدم المزودين الذين لديهم مصادقة صالحة، بدءًا من المزود الافتراضي الحالي، ثم المزودين المتبقين بترتيب أبجدي.

إذا فشل مزود، تتم تجربة المرشح التالي تلقائيًا. وإذا فشل جميع المرشحين، فسيتضمن الخطأ تفاصيل من كل محاولة.

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "google/veo-3.1-fast-generate-preview",
        fallbacks: ["runway/gen4.5", "qwen/wan2.6-t2v"],
      },
    },
  },
}
```

## ملاحظات المزود

| المزود | الملاحظات                                                                                                                                                 |
| ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Alibaba  | يستخدم نقطة النهاية غير المتزامنة DashScope/Model Studio. يجب أن تكون الصور ومقاطع الفيديو المرجعية عناوين URL بعيدة من نوع `http(s)`.                 |
| BytePlus | مرجع صورة واحدة فقط.                                                                                                                                     |
| ComfyUI  | تنفيذ محلي أو سحابي قائم على سير العمل. يدعم التحويل من نص إلى فيديو ومن صورة إلى فيديو عبر الرسم البياني المكوَّن.                                   |
| fal      | يستخدم تدفقًا قائمًا على قائمة انتظار للمهام الطويلة. مرجع صورة واحدة فقط.                                                                               |
| Google   | يستخدم Gemini/Veo. يدعم صورة مرجعية واحدة أو فيديو مرجعي واحد.                                                                                           |
| MiniMax  | مرجع صورة واحدة فقط.                                                                                                                                     |
| OpenAI   | يتم تمرير تجاوز `size` فقط. ويتم تجاهل تجاوزات النمط الأخرى (`aspectRatio` و`resolution` و`audio` و`watermark`) مع تحذير.                               |
| Qwen     | يستخدم خلفية DashScope نفسها مثل Alibaba. يجب أن تكون المدخلات المرجعية عناوين URL بعيدة من نوع `http(s)`؛ ويتم رفض الملفات المحلية مسبقًا.            |
| Runway   | يدعم الملفات المحلية عبر معرفات URI للبيانات. يتطلب التحويل من فيديو إلى فيديو `runway/gen4_aleph`. تعرض عمليات التشغيل النصية فقط نسب العرض إلى الارتفاع `16:9` و`9:16`. |
| Together | مرجع صورة واحدة فقط.                                                                                                                                     |
| Vydra    | يستخدم `https://www.vydra.ai/api/v1` مباشرةً لتجنب عمليات إعادة التوجيه التي تُسقط المصادقة. يأتي `veo3` مضمّنًا كنمط من نص إلى فيديو فقط؛ ويتطلب `kling` عنوان URL لصورة بعيدة. |
| xAI      | يدعم التحويل من نص إلى فيديو، ومن صورة إلى فيديو، وتدفقات تحرير/تمديد الفيديو البعيد.                                                                   |

## التكوين

اضبط نموذج إنشاء الفيديو الافتراضي في تكوين OpenClaw:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "qwen/wan2.6-t2v",
        fallbacks: ["qwen/wan2.6-r2v-flash"],
      },
    },
  },
}
```

أو عبر CLI:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "qwen/wan2.6-t2v"
```
