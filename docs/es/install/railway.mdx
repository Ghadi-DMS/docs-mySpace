Implementa OpenClaw en Railway con una plantilla de un clic y accede a él a través de la UI web de Control.
Esta es la ruta más sencilla de “sin terminal en el servidor”: Railway ejecuta el Gateway por ti.

## Lista rápida de verificación (personas usuarias nuevas)

1. Haz clic en **Deploy on Railway** (abajo).
2. Añade un **Volume** montado en `/data`.
3. Establece las **Variables** requeridas (al menos `OPENCLAW_GATEWAY_PORT` y `OPENCLAW_GATEWAY_TOKEN`).
4. Habilita **HTTP Proxy** en el puerto `8080`.
5. Abre `https://<your-railway-domain>/openclaw` y conéctate usando el secreto compartido configurado. Esta plantilla usa `OPENCLAW_GATEWAY_TOKEN` de forma predeterminada; si lo sustituyes por autenticación con contraseña, usa esa contraseña en su lugar.

## Implementación con un clic

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Deploy on Railway
</a>

Después de implementar, busca tu URL pública en **Railway → tu servicio → Settings → Domains**.

Railway hará una de estas dos cosas:

- te dará un dominio generado (a menudo `https://<something>.up.railway.app`), o
- usará tu dominio personalizado si has adjuntado uno.

Luego abre:

- `https://<your-railway-domain>/openclaw` — UI de Control

## Qué obtienes

- Gateway + UI de Control de OpenClaw alojados
- Almacenamiento persistente mediante Railway Volume (`/data`) para que `openclaw.json`,
  `auth-profiles.json` por agente, el estado de canal/proveedor, las sesiones y
  el espacio de trabajo sobrevivan a nuevas implementaciones

## Configuración requerida de Railway

### Red pública

Habilita **HTTP Proxy** para el servicio.

- Puerto: `8080`

### Volume (obligatorio)

Adjunta un volumen montado en:

- `/data`

### Variables

Establece estas variables en el servicio:

- `OPENCLAW_GATEWAY_PORT=8080` (obligatorio; debe coincidir con el puerto de Public Networking)
- `OPENCLAW_GATEWAY_TOKEN` (obligatorio; trátalo como un secreto de administración)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recomendado)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recomendado)

## Conectar un canal

Usa la UI de Control en `/openclaw` o ejecuta `openclaw onboard` mediante la shell de Railway para ver instrucciones de configuración de canales:

- [Telegram](/channels/telegram) (el más rápido: solo un token de bot)
- [Discord](/channels/discord)
- [Todos los canales](/channels)

## Copias de seguridad y migración

Exporta tu estado, configuración, perfiles de autenticación y espacio de trabajo:

```bash
openclaw backup create
```

Esto crea un archivo de copia de seguridad portable con el estado de OpenClaw más cualquier
espacio de trabajo configurado. Consulta [Backup](/cli/backup) para más detalles.
